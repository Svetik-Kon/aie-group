## 5. Analysis

- Устойчивость: В основной части эксперимента использовалось фиксированное разбиение train/test (random_state=42) 
для воспроизводимости и честного сравнения моделей. При смене random_state метрики могут немного меняться из-за
другого состава train/test, но при стратификации ожидаются умеренные колебания без изменения общего ранжирования
(ансамбли должны оставаться сильнее одиночного дерева и линейного baseline).

- Ошибки: 
Лучшая модель: GradientBoosting (tune max_depth)
Confusion matrix (test):
TN=3218, FP=100
FN=311, TP=871
По отчёту классификации:
для класса 0: precision 0.9119, recall 0.9699
для класса 1: precision 0.8970, recall 0.7369
Комментарий: модель хорошо “держит” нулевой класс (очень высокий recall), а по классу 1 основная доля ошибок
— FN (311 случаев), то есть часть положительных объектов модель всё ещё относит к 0. При этом precision по
классу 1 высокий (~0.897), то есть ложных срабатываний относительно мало.

- Интерпретация: Permutation importance для лучшей модели (критерий roc_auc, top-15) показал, что наиболее важные признаки:

f16 (0.0832 ± 0.0031) — самый сильный вклад (явный лидер), затем
f01 (0.0310 ± 0.0016), f07 (0.0182 ± 0.0012), далее группа признаков с меньшим вкладом:
f15, f19, f08, f23, f29, f12, f30, f18, f05, f13, f34 (примерно 0.006–0.014).

Интерпретация: признак f16 критически важен для разделения классов — при его перемешивании ROC-AUC падает сильнее всего.
Остальные признаки дают дополнительный вклад и, вероятно, участвуют во взаимодействиях/пороговых зависимостях, что ожидаемо
для синтетического датасета с нелинейными взаимодействиями и шума (ансамбли деревьев хорошо “ловят” такие эффекты).

## 6. Conclusion

- Одиночное дерево решений склонно к переобучению, поэтому важен контроль сложности (max_depth, min_samples_leaf), иначе качество на test ухудшается.

- Bagging-идея (RandomForest) заметно повышает качество относительно одиночного дерева за счёт снижения variance.

- Boosting (GradientBoosting) показал лучший результат на этом датасете, что согласуется с наличием нелинейных зависимостей и взаимодействий признаков.

- Для бинарной задачи с умеренным дисбалансом важно смотреть не только accuracy, но и F1 и особенно ROC-AUC.

- Честный ML-протокол: подбор гиперпараметров выполнялся только на train через CV, а test использовался только для финальной оценки, что снижает риск “подгонки под тест”.

- Permutation importance помог понять, какие признаки сильнее влияют на качество: у модели есть один “лидер” (f16) и группа признаков, дающих дополнительный вклад.
